Если вам нужно быстро перейти от пустого файла к отчёту, начните здесь. За несколько минут соберём
рабочий `main.typ`: титульный лист, исполнители, реферат, содержание, основные разделы, источники и
приложения. Пример можно использовать как каркас для лабораторной, курсовой или отчёта о НИР.

Не пытайтесь сразу идеально заполнить все поля. Сначала добейтесь, чтобы документ собирался, а уже
потом возвращайтесь к УДК, регистрационным номерам, грифам и локальным требованиям кафедры.

## Где писать код

Оба варианта поддерживают Live Preview: вы меняете `.typ`, а PDF обновляется рядом. Это сильно
снижает тревожность при первых правках — сразу видно, что именно изменилось в документе.

- [Typst App](https://typst.app/) работает в браузере и не требует установки.
- VS Code с [Tinymist](https://marketplace.visualstudio.com/items?itemName=myriad-dreamin.tinymist)
  удобен для локальной работы, Git и проектов с внешними файлами.

## Быстрый старт

Создайте файл `main.typ`. Первым делом нам нужно импортировать сам шаблон и его основные функции. Мы будем использовать актуальную версию из Typst Universe.

```typst
#import "@preview/modern-g7-32:0.2.0": gost, abstract, appendixes, enum-numbering
```

## Титульный лист и параметры

Шаблон запускается командой `#show: gost.with(...)`. Она применяет правила оформления ко всему
документу и создаёт титульный лист из переданных полей.

Добавьте этот блок сразу после импортов:

```typst
#show: gost.with(
  ministry: "Министерство науки и высшего образования Российской Федерации",
  organization: (
    full: "Национальный исследовательский томский политехнический университет",
    short: "ФГАОУ ВО НИ ТПУ",
  ),
  report-type: "Отчёт о лабораторной работе",
  subject: "Разработка сервиса маршрутизации заявок",
  city: "Томск",

  // В ГОСТе есть множество специфичных полей (УДК, номера регистрации).
  // Вы можете их добавить, а если они не нужны — просто не пишите.

  // manager: (name: "Иванов И.И.", position: "Доцент"),
  // ...
```

[Титульный лист и его настройки](./reference/title)

## Исполнители работы

Внутри той же функции `gost.with` есть параметр `performers`. Именно он отвечает за тех, кто выполнил работу. Добавьте его в вызов `gost.with`:

```typst
  performers: (
    (name: "И.О. Фамилия", position: "Студент группы 123"),
    // Если исполнитель один — он автоматически появится прямо на титульном листе.
    // Если их несколько — шаблон создаст отдельную страницу "СПИСОК ИСПОЛНИТЕЛЕЙ".
  ),
) // <-- Закрываем gost.with
```

[Список исполнителей и соисполнители](./reference/performers)

## Реферат

Следующий по ГОСТу структурный элемент — Реферат. В Typst вам не нужно вручную считать количество страниц, рисунков, таблиц и источников — функция `#abstract` сделает это за вас.

```typst
#abstract("маршрутизация", "графы", "api", "тестирование", "типизация")[
  В отчёте рассмотрена разработка сервиса маршрутизации заявок.
  Описаны модель графа, алгоритм поиска пути и результаты нагрузочного тестирования.
]
```

[Автоматизация реферата](./reference/abstract)

## Основная часть

Для создания разделов и подразделов используйте стандартный синтаксис Typst (`=`, `==`). Шаблон сам расставит правильные абзацные отступы, нумерацию (1.1, 1.2) и начнет новые главы с новой страницы.

```typst
= Проектирование сервиса
В этом разделе описана архитектура сервиса.

== Выбор архитектуры
Для решения задачи был выбран микросервисный подход. 
Как видно на рисунке @arch-scheme...

#figure(
  image("assets/scheme.png", width: 80%),
  caption: [Архитектура системы]
) <arch-scheme>
```
Внутри разделов вы можете использовать любые элементы: формулы, код, списки или таблицы. Шаблон автоматически приведет их к ГОСТу (отцентрирует, подпишет и пронумерует).

- [Правила основной части](./reference/body)
- [Оформление элементов](./reference/elements)

## Заключение и список источников

Заключение, как и Введение, не нумеруется. Сразу после него мы выводим список литературы.
Файл `references.bib` положите рядом с `main.typ`.

```typst
= Заключение
В ходе выполнения работы были достигнуты следующие результаты...

#bibliography("references.bib")
```

[Заключение](./reference/conclusion) и [Список использованных источников](./reference/references)

## Приложения

По ГОСТу приложения нумеруются буквами русского алфавита (исключая ё, з, й, о, ч, ъ, ы, ь). У них свои, независимые счётчики для рисунков и таблиц (Рисунок А.1).
Чтобы перевести документ в режим приложений, используйте `#show: appendixes`.

```typst
#show: appendixes

= Листинги исходного кода
В данном приложении представлен исходный код проекта.

#figure(
  ```python
  def hello_gost():
      print("ГОСТ 7.32-2017 соблюдён!")
  ```,
  caption:[Скрипт инициализации]
) <code-appendix>
```

[Оформление приложений](./reference/appendixes)

## Что автоматизирует шаблон

`modern-g7-32` берёт на себя оформление страницы, титульного листа, структурных заголовков,
содержания, статистики реферата, подписей рисунков и таблиц, библиографии и приложений. Вам всё
равно нужно написать содержательную часть отчёта, заполнить реальные реквизиты и проверить
локальные требования кафедры или организации.
