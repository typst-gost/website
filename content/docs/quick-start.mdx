Добро пожаловать! Если вы читаете это, значит, вам предстоит написать отчёт, курсовую или диссертацию, и вы (к счастью) решили не делать это в Word. 

В этом руководстве мы за 5 минут соберём готовый каркас вашего документа — от пустой страницы до приложений. Мы будем двигаться строго в хронологическом порядке, сверху вниз, так, как пишется код в самом файле `.typ`.

---

## Шаг 0. Где писать код?

Прежде чем начать, выберите удобный для вас инструмент. Оба варианта поддерживают **Live Preview** — вы печатаете код, и PDF-документ обновляется рядом за миллисекунды.

1. **В браузере:** [Typst App](https://typst.app/). Ничего не нужно устанавливать. Работает из коробки, поддерживает совместное редактирование (как Google Docs) и хранит проекты в облаке. Отличный выбор для старта.
2. **Локально:** Редактор **VS Code** + расширение [Tinymist](https://marketplace.visualstudio.com/items?itemName=myriad-dreamin.tinymist). Выбор продвинутых пользователей. Дает мощнейший автокомплит, оффлайн-работу, интеграцию с Git и синхронизацию курсора (кликаете по тексту в PDF — редактор перекидывает вас в нужную строку кода).

Определились? Тогда поехали!

## Шаг 1. Импорт шаблона

Создайте файл `main.typ`. Первым делом нам нужно импортировать сам шаблон и его основные функции. Мы будем использовать актуальную версию из Typst Universe.

```typst
#import "@preview/modern-g7-32:0.2.0": gost, abstract, appendixes, enum-numbering
```

## Шаг 2. Титульный лист и параметры

Вся магия шаблона запускается одной командой — `show: gost.with(...)`. Эта функция применяет правила оформления (шрифты, отступы, интервалы) ко всему тексту, а также автоматически генерирует титульный лист на основе переданных данных.

Добавьте этот блок сразу после импортов:

```typst
#show: gost.with(
  ministry: "Министерство науки и высшего образования Российской Федерации",
  organization: (
    full: "Национальный исследовательский томский политехнический университет",
    short: "ФГАОУ ВО НИ ТПУ",
  ),
  report-type: "Отчёт о лабораторной работе",
  subject: "Разработка системы навигации",
  city: "Томск",
  
  // В ГОСТе есть множество специфичных полей (УДК, номера регистрации).
  // Вы можете их добавить, а если они не нужны — просто не пишите.
  
  // manager: (name: "Иванов И.И.", position: "Доцент"),
  // ...
```
> 📘 **Узнать больше:**[Титульный лист и его настройки](./reference/title)

## Шаг 3. Исполнители работы

Внутри той же функции `gost.with` есть параметр `performers`. Именно он отвечает за тех, кто выполнил работу. Добавьте его в вызов `gost.with`:

```typst
  performers: (
    (name: "И.О. Фамилия", position: "Студент группы 123"),
    // Если исполнитель один — он автоматически появится прямо на титульном листе.
    // Если их несколько — шаблон создаст отдельную страницу "СПИСОК ИСПОЛНИТЕЛЕЙ" (ГОСТ 7.32, п. 5.2).
  ),
) // <-- Закрываем gost.with
```
> 📘 **Узнать больше:**[Список исполнителей и соисполнители](./reference/performers)

## Шаг 4. Реферат

Следующий по ГОСТу структурный элемент — Реферат. В Typst вам не нужно вручную считать количество страниц, рисунков, таблиц и источников — функция `#abstract` сделает это за вас.

```typst
#abstract("навигация", "алгоритмы", "графы")[
  В данной работе рассматриваются вопросы проектирования систем навигации.
  Разработан алгоритм обхода препятствий...
]
```
> 📘 **Узнать больше:**[Автоматизация реферата](./reference/abstract)

## Шаг 5. Содержание и Введение

Тут всё максимально просто. Вызываем встроенную функцию `#outline()` для сборки оглавления, а затем начинаем писать текст. Шаблон уже знает, что `ВВЕДЕНИЕ` не нумеруется и должно быть оформлено заглавными буквами.

## Шаг 6. Основная часть (Текст, рисунки, таблицы)

Для создания разделов и подразделов используйте стандартный синтаксис Typst (`=`, `==`). Шаблон сам расставит правильные абзацные отступы, нумерацию (1.1, 1.2) и начнет новые главы с новой страницы.

```typst
= Проектирование системы
В этом разделе мы спроектируем систему. 

== Выбор архитектуры
Для решения задачи был выбран микросервисный подход. 
Как видно на рисунке @arch-scheme...

#figure(
  image("assets/scheme.png", width: 80%),
  caption: [Архитектура системы]
) <arch-scheme>
```
Внутри разделов вы можете использовать любые элементы: формулы, код, списки или таблицы. Шаблон автоматически приведет их к ГОСТу (отцентрирует, подпишет и пронумерует).

> 📘 **Узнать больше:** 
> * [Правила основной части](./reference/body)
> *[Оформление элементов (Рисунки, Таблицы, Формулы, Код)](./reference/elements)

## Шаг 7. Заключение и Список источников

Заключение, как и Введение, не нумеруется. Сразу после него мы выводим список литературы.
Вам достаточно закинуть файл `references.bib` в папку с проектом, и шаблон сам оформит всё по ГОСТ Р 7.0.5-2008.

```typst
= Заключение
В ходе выполнения работы были достигнуты следующие результаты...

#bibliography("references.bib")
```
> 📘 **Узнать больше:** [Заключение](./reference/conclusion) и [Список использованных источников](./reference/references)

## Шаг 8. Приложения

По ГОСТу приложения нумеруются буквами русского алфавита (исключая ё, з, й, о, ч, ъ, ы, ь). У них свои, независимые счётчики для рисунков и таблиц (Рисунок А.1).
Чтобы перевести документ в режим приложений, используйте `#show: appendixes`.

```typst
#show: appendixes

= Листинги исходного кода
В данном приложении представлен исходный код проекта.

#figure(
  ```python
  def hello_gost():
      print("ГОСТ 7.32-2017 соблюдён!")
  ```,
  caption:[Скрипт инициализации]
) <code-appendix>
```
> 📘 **Узнать больше:**[Оформление приложений](./reference/appendixes)
